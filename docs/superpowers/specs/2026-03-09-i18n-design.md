# i18n 国际化设计文档

**日期：** 2026-03-09
**范围：** 前端 (React) + 后端 (Go)
**首期支持语言：** English (en)、简体中文 (zh)

---

## 目标

为 robobee 管理界面添加国际化支持，用户可在英文和简体中文之间切换，语言偏好持久化到 localStorage，前端请求携带 `Accept-Language` header，后端据此返回本地化错误信息。

---

## 整体架构

```
前端 (react-i18next)
├── web/src/i18n.ts              # i18next 初始化配置
├── web/src/locales/
│   ├── en.json                  # 英文翻译
│   └── zh.json                  # 简体中文翻译
├── web/src/components/
│   └── language-switcher.tsx    # 语言切换按钮（嵌入 nav.tsx）
└── 所有页面/组件使用 useTranslation hook

后端 (go-i18n)
├── locales/
│   ├── en.toml                  # 英文错误信息
│   └── zh.toml                  # 中文错误信息
├── server/middleware/
│   └── i18n.go                  # 解析 Accept-Language，注入 Localizer 到 context
└── 各 handler 使用 localizer.Localize() 返回本地化错误

通信协议
└── 前端每次 API 请求携带 Accept-Language: en 或 Accept-Language: zh header
```

---

## 前端设计

### 依赖
- `react-i18next` + `i18next`

### i18n 初始化 (`web/src/i18n.ts`)
- 启动时读取 `localStorage.language`，默认 `en`
- fallback 语言为 `en`
- 翻译文件静态 import（项目规模小，不需懒加载）

### 翻译文件结构
按页面/功能分组：
```json
{
  "nav": { "workers": "Workers", "executions": "Executions" },
  "workers": { "title": "Workers", "create": "Create Worker" },
  "common": { "save": "Save", "cancel": "Cancel", "delete": "Delete" }
}
```

### 语言切换器 (`web/src/components/language-switcher.tsx`)
- 两个 tab 按钮：`EN` / `中文`，嵌入导航栏右侧
- 切换时：更新 i18next 语言 + 写入 localStorage + 更新 API client header

### API Client (`web/src/lib/api.ts`)
- 所有请求默认带 `Accept-Language` header，值从 i18next 当前语言读取

### 组件改造
- 所有硬编码字符串替换为 `t('key')` 调用
- 涉及文件：`nav.tsx`、`layout.tsx`、所有 `pages/` 页面组件

## 后端设计

### 依赖
- `github.com/nicksnyder/go-i18n/v2`

### 翻译文件 (`locales/en.toml`, `locales/zh.toml`)
```toml
# en.toml 示例
[WorkerNotFound]
other = "Worker not found"

[InvalidCronExpression]
other = "Invalid cron expression: {{.Expr}}"
```

### i18n 中间件 (`server/middleware/i18n.go`)
- 解析 `Accept-Language` header
- 创建 `*i18n.Localizer`，注入到 Gin context
- 不支持的语言自动 fallback 到 `en`

### 错误返回改造
- 各 handler 从 Gin context 取出 localizer
- 通过 `localizer.Localize()` 获取本地化错误文本后返回
- **不翻译**正常业务数据（worker 名称、日志内容等），只翻译系统产生的错误信息

---

## 扩展性

- 新增语言只需添加翻译文件，无需改动代码
- 前端类型安全翻译键（i18next TypeScript 插件）可作为后续优化项
